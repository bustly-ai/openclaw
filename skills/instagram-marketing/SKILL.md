# Instagram Marketing Generator

## Overview

Instagram Marketing Generator produces operator-ready outputs that can be sent, reviewed, or handed off without another rewrite loop.

## When to Use

- The request is fundamentally about content production work, and the operator needs a usable output.
- The team could act immediately if Instagram Marketing Generator returns a crisp verdict, deliverable set, and next-step list.

## Do Not Use When

- Do not pretend a mutation, publish, render, or API call already happened when the skill only provides planning or analysis.
- Do not keep this skill active once the task clearly belongs to a narrower adjacent skill.
- Do not invent product claims, proof points, legal language, or performance numbers that the user did not provide.
- Do not stop at vague ideas when the output should be copy-ready or handoff-ready.

## Required Inputs

- source material or transcript
- target channel / format
- audience + CTA
- brand guardrails
- must-say points
- forbidden claims or no-go tone
- examples of acceptable output if available
- product angle or campaign theme
- which Instagram surfaces are needed (feed / story / reel)
- visual direction or brand style constraints

## Deliverables

- one-line verdict or executive summary
- evidence or rationale table
- priority-ranked actions or decisions
- risks and recommended next steps
- copy-ready or handoff-ready output
- brief QA checklist
- missing-fact or approval note
- asset, script, or brief structure

## Workflow

1. Clarify which Instagram surfaces matter most: feed, story, reel, or a coordinated package.
2. Build the content package with clear role separation instead of reusing one draft everywhere.
3. Keep caption logic, visual direction, and CTA behavior platform-native rather than generic cross-platform copy.
4. End with posting notes and the first thing to test if engagement is weak.

## Output Format

1. Short verdict or framing note
2. Context / assumptions
3. Copy-ready or handoff-ready output
4. QA checklist / risk notes
5. Recommended next step

## Boundary and Routing

- Keep the skill focused on its core decision or output and hand off once the task narrows to execution.

## Quality Rules

- Separate observed facts, user-provided facts, and assumptions.
- Do not invent claims, metrics, specs, or proof that were not provided or verified.
- Prefer the shortest useful answer that an operator can actually use.
- Make the draft usable without another rewrite loop.
- Downgrade risky claims instead of polishing them into something unsafe.

## Instagram Package Rules

- Differentiate feed, story, and reel behavior rather than reusing one draft everywhere.
- Include CTA placement, caption logic, and a hashtag / posting-note strategy only if it materially helps the operator.

## Missing Data Protocol

- Ask for the smallest missing set that would materially change the answer.
- If the core blocker is missing, stop short of a hard verdict and say exactly what is needed.
- If only secondary inputs are missing, continue in checklist or hypothesis mode and label every assumption.
- End low-confidence outputs with the single most valuable next data request instead of hiding uncertainty.

## Validation Loop

- Recheck: publish quality / engagement proxy / reuse readiness.
- Review window: after review, publish, or the first live use of the output.
- Define what improvement, no-change, and failure look like before closing the task.
- If direct analytics access is missing, replace the metric with a manual checkpoint and label that downgrade.

## Example

**Scenario:** 一家DTC家居品牌，主打竹纤维床品套装，售价$89，准备在Instagram发起夏季新品上市活动，需要一套完整的内容包：Feed主图帖子+Carousel教育内容+Story促销+Reel脚本框架，目标是7天内带来200个新粉丝和50次加购。

**Expected response shape:**
- Start with the conclusion or verdict.
- Show the evidence or assumptions used.
- Give the operator-ready actions or draft output.
- End with risks and next steps.
